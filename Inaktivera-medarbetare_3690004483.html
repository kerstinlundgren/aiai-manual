Nya Aiai manual : Inaktivera medarbetare

Created by Kerstin Lundgren on Jul 11, 2025

Att inaktivera en medarbetare innebär att personen tillfälligt stängs av från systemets centrala delar utan att tas bort helt. Funktionen gör det möjligt att återaktivera personen vid behov och samtidigt säkerställa att relaterad historik, rapporter och kopplingar bibehålls.

Syfte

Funktionen används till exempel för att hantera:

  • Semestervikarier eller andra tillfälligt anställda

  • Medarbetare som inte längre är aktuella men som kan återvända

För dig som arbetat i Aiai Senior motsvarar inaktivering av medarbetare att flytta denne till användarbanken.

Inaktivera en medarbetare

En medarbetare kan inaktiveras på två sätt:

  1. Via medarbetarens profilsida

    1. Gå till fliken Inställningar i medarbetarens profil.

    2. I kortet Status klickar du på Inaktivera.

    3. En bekräftelsedialog visas. Bekräfta åtgärden för att inaktivera medarbetaren.

  1. Via medarbetarlistan

    1. Gå till fliken Medarbetare

    2. Markera en eller flera personer i listan.

    3. Klicka på Inaktivera och bekräfta i dialogen

Filtrera på aktiv och inaktiv

Som användare kan du filtrera medarbetarlistan på aktiva / inaktiva medarbetare. Detta gör du genom att:

  1. Klicka på Filter

  2. För parametern Kolumner, välj Status

  3. För Operator, välj är (förvalt)

  4. För Värde, välj Aktiv.

→ Om du istället vill filtrera på inaktiva medarbetare, välj Inaktiv i Värde

Vad händer vid inaktivering?

När en medarbetare inaktiveras:

  • Personen kan inte längre logga in i Aiai.

  • Personen försvinner ur alla vyer där endast aktiva medarbetare visas:

    • Kontextväljaren

    • Widgetar

    • Ledighetsansökan

    • Kopplingslistor (från kontexten Enhet/Brukare → Medarbetare)

  • Personen syns fortsatt i:

    • Medarbetarlistan (med status "Inaktiv")

      • Medarbetarlistan kan filtreras på enbart aktiva, så att de inaktiva inte visas

    • Tidrapport, schemakontroll, lönedata, FK3059

    • Scheman (historiska pass får etiketten “inaktiv”)

    • Kopplingslistor (från kontexten Medarbetare → Brukare)

Märkning i systemet

  • I exempelvis journalanteckningar, avtal, chatt och loggar visas “[Inaktiverad]” efter personens namn.

  • Inaktiveringsdatum visas i personens profil under Status.

Schema

  • Alla framtida pass tas bort eller blir obemannade:

    • Medarbetarschema: pass tas bort

    • Enhetsschema: pass blir obemannade

    • Schemamall: pass blir obemannade

  • Schemat blir read-only; inga ändringar kan göras i historiska pass där personen varit schemalagd.

Återaktivera medarbetare

  1. Gå till medarbetarens profil eller markera personen i medarbetarlistan.

  2. Klicka på Återaktivera.

  3. Bekräfta åtgärden i dialogen.

Vid återaktivering:

  • Medarbetaren återfår alla tidigare kopplingar och rättigheter.

  • “[Inaktiverad]” tas bort från personuppgifter.

  • Åtkomst till Aiai återställs.

Rättigheter

  • Endast administratörer och huvudadministratörer kan inaktivera eller återaktivera en medarbetare.

  • En användare kan inte inaktivera sig själv, oavsett roll

  • Om en inaktiverad medarbetare är angiven som kontaktperson till Försäkringskassan, byts denna automatiskt ut till företagets standardkontaktperson.

Övrig information

  • Användare som varit inaktiva hela månaden räknas inte med i fakturering, användarstatistik eller eventuella beräkningar i licensmodellen.

  • All information (rapporter, kopplingar etc.) sparas och återställs automatiskt vid återaktivering.

  • Inaktivering påverkar inte historiska rapporter eller insikter där användaren är relevant.

I kommande version av Aiai kommer det även att vara möjligt att göra ett “hårt” borttag av användare, d.v.s. radera denne helt.
