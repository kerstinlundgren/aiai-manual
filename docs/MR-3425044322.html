Status på arbetspass - reusable

Status på arbetspass

Arbetspass kan ha något av nedanstående tre statusar. Du ändrar status på ett eller flera arbetspass genom att markera dem och klicka på de tre prickarna bredvid “kugghjulet”. Du kan även ändra status på alla pass inom en vald tidsperiod genom att klicka på de tre prickarna utan att ha valt något arbetspass. När du då väljer en status genom att klicka på den får du upp en datumväljare och kan definiera den tidsperiod inom vilken du vill ändra status på arbetspass.

Planerat

Ett arbetspass med status planerat syns enbart för användare med rättigheter att lägga schema, d.v.s. huvudadministratör, administratör och medarbetsledare. Syftet med denna status är att du som schemaläggare ska kunna arbeta och laborera med arbetspass och olika schemalösningar utan påverkan för medarbetarna.

Publicerat

Ett arbetspass med status publicerat är synligt för medarbetaren samt för övriga medarbetare i enheten, om alternativen “Medarbetare kan se varandras arbetspass” är valda.

Beroende på verksamhetens rutiner kan ett publicerat schema betraktas som “föreslaget” fall där verksamheten arbetar med olika typer av önskescheman, eller där man vill föreslå ett framtida schema men tydliggöra att det ännu inte är gällande. Föreslagna arbetspass visualiseras med en streckad linje i vänstersidan.

Låst

Låsta arbetspass är synliga för medarbetare och är att betrakta som ett färdigt schema. För verksamheter som arbetar med schemaperioder som ska låsas/fastslås inom en överenskommen tid innan de börjar gälla är denna funktion lämplig. Låsta pass går inte att ändra avseende start- och sluttid, aktiviteter och vald medarbetare utan att först låsa upp dem. Syftet här är att inte råka ändra saker i ett arbetspass. Det går givetvis bra att skriva anteckningar i låsta pass.

Fastställda aktiviteter

Låsta arbetspass visualiseras med en heldragen linje i vänstersidan. När ett arbetspass låses blir dess aktiviteter fastställda (de visualiseras med blå text) vilket innebär att även om du som användare ändrar något i bemanningsbehovet eller defaultinställningarna för tidtyp, aktivitetstyp eller finansiär så kommer låsta arbetspass inte att påverkas.

Om du “backar” status på arbetspass som varit låsta kommer dess aktiviteter fortfarande att vara fastställda. Om du vill återgå till de “flytande” aktiviteterna som automatiskt kommer från schemainställningar eller bemanningsbehov, tar du enkelt bort de fastställda aktiviteterna genom att klicka på de tre prickarna vid dessa blåfärgade aktiviteter och väljer “Radera” varpå arbetspasset åter automatiskt kommer att populeras med de flytande aktiviteterna.

Arbetspass måste vara låsta för att visas i tidrapporten, därför låses pass per automatik senast vid den tidpunkt då de startar.

Automatiserade regler för statusändring

Under “Företagsinställningar” → “Schema” kan du som huvudadministratör sätta upp regler för hur många dagar i förväg innan ett arbetspass infaller, som det automatiskt ska få status publicerat respektive låst. De inställningar som är valda här blir också förvalda för alla nya organisationsenheter som skapas.

Motsvarande inställningar kan göras för respektive organisationsenhet, i det fall andra regler ska gälla i en viss organisationsenhet. Dessa inställningar görs då under vald organisationsenhets schema → “Schemainställningar”

Ett arbetspass måste vara publicerat för att kunna låsas, därmed kan värdet för publicerade pass inte vara mindre än det för låsta pass. Värdena för publicerade och låsta pass kan vara samma, i det fallet går passen direkt från status planerat till låst.

Ett pass måste vara låst för att visas i tidrapporten, därmed är det minsta möjliga värde att sätta för automatisk låsning 1 dag innan passet infaller.

Statusförändringar på publicerade och låsta pass

Om du som användare ändrar status på ett publicerat eller låst pass efter att tiden för automatisk statusförändring passerat, t.ex. i det fall du behöver göra förändringar på ett redan låst pass, kommer passet att återfå sin ursprungliga status 30 minuter efter att du senast hanterat det.

Jobbet för statusförändring i enöghet med (publicera + låsa) körs var 30:de minut, och tar med alla pass inom den automatiserade perioden och ändrar dessa statusar till den korrekta. Pass som hanterats av användare inom en tidsperiod av 30 minuter statusändras inte, detta för att passen inte plötsligt ska ändra status medan du som användare arbetar med dem.
